Linux-Shell-Introduction

What is the Linux-Shell?

The linux-shell is a component of the "gnu/linux"-os (operating-system), which is the outermost part of the os and could therefore be called the: "interface" of the os.

"gnu/linux" is normally composed of the following parts:

  1. Hardware
  2. Kernel
  3. Shell
  4. CLI/GUI
  5. Applications

Hardware and kernel can only be accessed through the interface of the shell.

And as shown through the seventh list-element, the shell can either be accessed through a cli (command-line-interface), or a gui (graphical user-interface)

One can configure the "gnu/linux"-os to boot into a cli, but it is more common to boot into a gui and then use a terminal-emulator-application in order to access the cli-shell; this is also the way shown in this introduction.

Accessing the Linux-Shell.

This introduction focuses on beginners and therefore is going to assume that it's reader is using a standard "gnu/linux"-desktop-environment, which ships with a terminal-emulator already installed (like GNOME on Ubuntu).

Using the System-Search

In order to open a terminal-emulator via the system-search-tool, press the: "Meta"-key (colloquial often called the: "Windows"-key, or on a Mac-keyboard, it should be the: "option"-key").

This should open the system-search.

Then type in something like: "ter", "term" or "terminal".

Which should result in the os-search finding an application with terminal in its name; open it.

Using the Start-/Applications-menu

Open the start-/applications-menu through the designated button in your task-bar.

Search for a program with: "terminal" in its name and open it.

(If your desktop-environment uses preconfigured folders in the start-/applications-menu, search in the: "tools"-, or "accessorizes"-folder.)

Navigating the Linux-Shell

Before every command you type in at the position of the caret, is a prefix in the following format: "[user]@[computer-name]".

Basic Commands

The following commands are the most elemental commands which are needed to explore and learn the linux-shell.

For the start they are all you really need, and they are also going to serve as examples for further explanations.

ls

"ls" stands for: "list".

It is used in the following way: "ls".

In other words, you can just type it in the command-line, press return, and it is going to list all non-hidden files and folders in the current working-directory (the directory your cli is focused on right now).

You can use this command in order to see where you can go in the directory-hierarchy and which files are available to you.

pwd

"pwd" stands for: "print working directory".

It is used in the following way: "pwd"

It prints the absolute path (the path from your root-directory to the directory you are currently in), into the console.

This command may be used to navigate the directory-hierarchy, by being able to know where your current working-directory is positioned.

cd

"cd" stands for: "change directory".

It is used in the following way: "cd [dirname]

So if you type: "ls", you are going to see all directories you can change in and then using: "cd [dirname], you can change your working directory to it.

Therefore, it can be used to do the actual navigating.

Getting Help

While exploring the shell, it is more important to know how to gather unknown information about the shell, than knowing how to use the shell 100% of the time.

What this means is, it is important to know how to use the shell's help-system and external sources, like web-searches.

Don't think you have to learn all the commands and remember them at all time, this is not how it works.

-h and --help

Nearly every shell-command has a so called: "option", called: "-h" and/or: "--help".

If you type it like this: "[command] --help", you are going to get a shell-printout of the usage, description and most important options of the command.

(Options are modifiers of a command's functionality, which are initiated by typing them after the name of the command, like described above with the help-option.)

man

"man" unlike: "-h" and "--help" is its own command and takes a name of another command as an argument (like "cd" takes a directory-name as an argument), in order to display a detailed man-page

This man-page is displayed in a separate terminal-view and lets you scroll through it with the arrow-keys.

You can leave this view by pressing the: "q"-key.

You can enter search-queries by typing: "/", typing in the word you want to search for and then pressing return.

Web-Searching

As already mentioned above, there is absolutely no shame in using web-searches if you don't know a command, or even if you can't completely decipher the: "--help"- or man-page-printouts of a command you do know.

Even people who use the linux-shell a lot don't know all commands and/or all their options.

So if, after this introduction, you would want to know how to creat a file, or a directory; you could search the following in your search-engine of trust: "linux shell create file" or "linux shell create directory".

Which would then lead you to websites which are probably going to show you that the: "touch"-command is going to do the trick here for files and the "mkdir"-command for directories.

Conclusion

You are know equipped with the following:

  1. Basic knowledge about the linux-shell.
  2. Knowledge of how to access the linux-shell.
  3. Knowledge of the most elemental commands to navigate the shell.
  4. Knowledge of the linux-shell's help-system and what options are.

These basics are going to give you a good elemental understanding to build on, because you now know how to navigate the shell, and you know how to use it's help-system if you want to figure out how to do something.
